# 连续劳务报酬个税计算器 Spec

## Why
随着《国家税务总局关于互联网平台企业为平台内从业人员办理扣缴申报、代办申报若干事项的公告》（2025年第16号）的实施，自2025年10月1日起，平台从业人员（外卖骑手、网约车司机、直播主播等）的劳务报酬所得适用累计预扣法计算个税。目前缺乏一款直观、跨平台的工具，帮助从业者快速测算税前/税后收入，理解不同月份收入对税负的累计影响。

## What Changes
- 构建基于 Rust + Tauri 2 的跨平台桌面应用（macOS / Windows）
- 实现连续劳务报酬累计预扣法的正向计算（税前→税后）
- 实现连续劳务报酬累计预扣法的反向推算（税后→税前）
- 支持多月份不同收入的累计计算与可视化展示
- 实现一般劳务报酬按次预扣法的对比计算
- 实现年度汇算清缴的模拟计算
- 精美的金融级 UI 设计，支持深色/浅色主题

## Impact
- Affected code: 全新项目，无既有代码影响
- 技术栈: Rust (核心计算引擎) + Tauri 2 (跨平台框架) + React + TypeScript + Tailwind CSS (前端UI)

## ADDED Requirements

### Requirement: 核心计算引擎 (Rust)
系统 SHALL 提供基于 Rust 实现的连续劳务报酬个税计算引擎，作为 Tauri 后端核心模块。

#### Scenario: 连续劳务报酬累计预扣法正向计算
- **WHEN** 用户输入多个月的税前劳务报酬收入
- **THEN** 系统按照累计预扣法计算每月应预扣预缴税额和税后收入
- **计算公式**: 本期应预扣预缴税额 = (累计收入 - 累计费用 - 累计减除费用) × 预扣率 - 速算扣除数 - 累计已预扣预缴税额
- **累计费用** = 累计收入 × 20%
- **累计减除费用** = 5000元/月 × 连续取得劳务报酬的月份数
- **预扣率表** (预扣率表一):

| 级数 | 累计预扣预缴应纳税所得额 | 预扣率(%) | 速算扣除数 |
|------|--------------------------|-----------|------------|
| 1 | 不超过36000元 | 3 | 0 |
| 2 | 超过36000元至144000元 | 10 | 2520 |
| 3 | 超过144000元至300000元 | 20 | 16920 |
| 4 | 超过300000元至420000元 | 25 | 31920 |
| 5 | 超过420000元至660000元 | 30 | 52920 |
| 6 | 超过660000元至960000元 | 35 | 85920 |
| 7 | 超过960000元 | 45 | 181920 |

#### Scenario: 连续劳务报酬累计预扣法反向推算
- **WHEN** 用户输入多个月的期望税后收入
- **THEN** 系统通过数值迭代法反推每月所需的税前收入
- **约束**: 反推结果与正向计算的误差不超过 0.01 元

#### Scenario: 一般劳务报酬按次预扣法计算
- **WHEN** 用户选择一般劳务报酬模式并输入单次收入
- **THEN** 系统按照预扣率表二计算应预扣预缴税额
- **减除费用**: 收入≤4000元时减除800元，收入>4000元时减除20%
- **预扣率表** (预扣率表二):

| 级数 | 预扣预缴应纳税所得额 | 预扣率(%) | 速算扣除数 |
|------|----------------------|-----------|------------|
| 1 | 不超过20000元 | 20 | 0 |
| 2 | 超过20000元至50000元 | 30 | 2000 |
| 3 | 超过50000元 | 40 | 7000 |

#### Scenario: 年度汇算清缴模拟计算
- **WHEN** 用户输入全年劳务报酬收入及专项扣除信息
- **THEN** 系统计算全年综合所得应纳税额，并与预扣预缴税额对比，得出应补/退税额
- **全年应纳税所得额** = 劳务报酬收入额(收入×80%) - 60000元 - 专项扣除 - 专项附加扣除 - 其他法定扣除
- **适用税率表**: 综合所得税率表（与预扣率表一相同）

### Requirement: 多月份收入管理
系统 SHALL 支持用户录入和管理1-12个月的不同劳务报酬收入数据。

### Requirement: 跨平台桌面应用
系统 SHALL 基于 Tauri 2 构建，支持 macOS 和 Windows 双平台运行。

### Requirement: 精美金融级 UI 设计
系统 SHALL 提供专业、精美的金融级用户界面，符合 fintech 产品设计规范。
- 主色调采用深蓝 (#1E3A5F) + 翡翠绿 (#10B981) 的专业金融配色
- 辅助色使用暖金色 (#F59E0B) 用于强调和警示
- 支持深色/浅色主题切换

### Requirement: 数据导出
系统 SHALL 支持将计算结果导出为 CSV 文件。

### Requirement: 输入验证与错误处理
系统 SHALL 对用户输入进行严格验证。
